The Platform

One pipeline.
Ten modules.

Every module reads and writes to a shared BlueprintJSON — the single source of truth. Nothing is isolated. Everything compounds.

MODULE 01
✦
Memory-to-Masterpiece AI
A client shares a memory. This module converts raw narrative into structured EmotionScore arrays, Fauxever Memories glyphs, and a fully resolved BlueprintIntent object ready for the pipeline.
Memory text → EmotionScore[] + BlueprintIntent
MODULE 02
🎨
Emotional Floral Mapper
Emotion chords and glyphs drive every floral selection. The mapper uses tier buckets, weighted randomness, and diversity constraints — never a scoring system that repeats the same flowers.
EmotionScore[] → FloralRecipeDraft + palette
MODULE 03
📐
Anchor Point Visualizer
600×600px canvas, 25px/inch scale. Every placement is a polar coordinate (rNorm, theta). Focal clusters at clock arcs. Greenery distributed by golden angle (137.5°). Stems rotate tangentially — always perpendicular to the wreath.
BlueprintIntent → visual canvas + coordinate JSON
MODULE 04
💬
AI Florist Assistant
The florist explains every placement decision in real time. Ask why eucalyptus anchors a grief design, what z-index a garden rose cluster should carry, or how golden angle prevents spacing repetition.
Any element → floriography + placement rationale
MODULE 05
📄
Blueprint JSON Generator
Export a manufacturing-ready JSON with full polar coordinates, stem counts, SKU references, z-index layers, and cluster metadata. The document any builder needs to construct the wreath precisely.
BlueprintLayout → manufacturing JSON
MODULE 06
📸
Floral Photo Analyzer
Upload a stock photo or catalog image of any floral element. The analyzer extracts FloralElement data — estimated head size, color family, texture category, and FFLGA role classification.
Photo → FloralElement (size, color, texture, role)
MODULE 07
📖
Wreath Build Guide Creator
Transforms a finalized blueprint into a beautiful PDF containing the client's story, the floral recipe with symbolism notes, the placement blueprint, and step-by-step construction instructions.
Blueprint + recipe → Story/Recipe/Blueprint PDF
MODULE 08
🎭
Bloomy-Style Design Studio
A React drag-drop editor that hosts all modules in a single workspace. The canvas is the center. Every plugin implements WreathElementPlugin — the canvas never knows specific SKUs, only the contract.
Full design workspace with plugin architecture
MODULE 09
⚖️
Symmetry Engine
Enforces bilateral or radial balance using visual weight calculation per quadrant (role × head size × saturation). Target: focal quadrant 1.2–1.4× average. All others within ±20%.
BlueprintLayout → balanced BlueprintLayout
MODULE 10
⭐
Aesthetic Fitness Evaluator
A GNN-based scoring system that evaluates every design against composition rules. Returns a fitness score, quadrant weight breakdown, and prioritized improvement suggestions before final export.
BlueprintLayout → score + improvement list

Design System

Six principles
that don't bend.

Every module decision traces back to these foundational rules. They are the constitution of the ecosystem.

PRINCIPLE 01
One source of truth
All modules read and write BlueprintJSON. No ad-hoc state, no prop drilling, no fragmented data stores. The blueprint is the contract.
PRINCIPLE 02
Emotion drives geometry
Every placement decision traces back to the client's emotions and glyphs. The math is always in service of meaning — never the other way around.
PRINCIPLE 03
Non-scoring selection
Tier buckets and weighted randomness instead of numeric scoring. Scores cause repetition. Randomized diversity constraints produce living, unrepeatable designs.
PRINCIPLE 04
Polar coordinates only
All positions use (rNorm, theta). Canvas: 600×600px, 25px/in. Stems rotate at angleDeg + 90° — always tangential, always perpendicular to the wreath's curvature.
PRINCIPLE 05
FFLGA roles always
Every inventory item is classified as Focal, Filler, Line, Greenery, or Accent before it enters any algorithm. Classification is non-negotiable — it governs placement, z-index, and stem counts.
PRINCIPLE 06
Plugin architecture
Every element implements WreathElementPlugin. The canvas never knows about specific flower types — only the interface contract. New botanicals require zero core changes.
