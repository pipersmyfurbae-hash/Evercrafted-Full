Primary User
Wreath makers, faux floral artists, and florists who sell on Etsy or run custom order businesses. They have existing inventory, client requests with emotional language, and no structured design system to translate brief → blueprint → listing.
Core Problem We Solve
The translation gap between "I want something that feels like a cozy fall morning" and a physically buildable wreath with exact stem counts, polar placements, render prompts, and a live Etsy listing. That gap takes skilled makers 2–4 hours per design. Evercrafted collapses it to minutes.

Competitive Defensibility
Evercrafted's Moat
Proprietary Schema
EC_CANON_V1 / EC_WR_V2 is a domain-specific blueprint format. Competitors would need to reverse-engineer 12+ composition formulas, QACS coordinate system, and odd-cluster placement logic.
Emotion Engine
Valence-Arousal → color palette → shape language → bouba/kiki mapping → floral selection is a multi-step translation chain that took iterations to calibrate. Not replicable quickly.
Render Discipline
Anti-fresh render rules (no dew, no wild scatter, grapevine integration cues, silk surface properties) are a trained prompt system. Generic prompt tools produce fresh-flower results.
Genome System (WGS)
Design-as-compact-string allows sharing, mutation, breeding, and versioning of wreath designs. This is a novel concept with no analog in the floral or craft SaaS space.
Inventory Intelligence
The platform only uses what a maker actually has. Hallucinated substitutions are flagged and blocked by rule. This constraint-aware design is a trust signal competitors can't easily match.
Niche Network Effects
As blueprints accumulate in the marketplace and more makers validate designs, the corpus of EC_CANON blueprints becomes a flywheel. The more designs, the more valuable the platform.

The Core Pipeline — Emotion In, Design Package Out
Quick Win · MVP · Ship-First · 4–6 Week Build
BUILD NOW Web App
A single-session web tool where a maker enters their client's brief and inventory list, and receives a complete design package: emotion profile, blueprint JSON, render prompt, step-by-step build guide, and Etsy listing copy. No account required in v1. No database. No subscriptions. Just the pipeline working end-to-end. This is the quick win — it proves the system and gets real users generating real designs.

  • Inventory Input — paste or type inventory list (CSV or plain text). System categorizes into base / greenery / focal / filler / accent.
  • Wreath Size Selector — 14", 18", 24", 30" with auto stem count ranges populated.
  • Claude API Pipeline — single orchestrated call runs all 7 stages and returns structured JSON package.
  • Blueprint Viewer — renders the clock-position placement map as an interactive SVG (basic; no full QACS canvas yet).
  • Render Prompt Output — formatted, copyable Midjourney-ready prompt with all anti-fresh rules applied.
  • Build Guide — numbered steps by layer with attachment method and clock positions.
  • Etsy Listing Copy — title, description, bullets, 13 tags, price suggestion. One-click copy each section.
  • Download Package — full design package as a single JSON file and/or formatted PDF-ready HTML.

Accounts, Inventory Library & Blueprint History
Retention Layer · Supabase Integration · 6–8 Weeks After Phase 1
PLAN NEXT Auth Required
Add user accounts so makers can save their inventory, reuse it across sessions, and access their blueprint history. This is the retention unlock — once someone has their inventory saved and blueprints stored, switching cost increases significantly. Supabase handles auth + database with RLS.
  • Supabase Auth — email/password + magic link. Row-level security so each user only sees their data.
  • Inventory Library — saved, reusable inventory lists. Tag items with category, color, quantity. Persist across sessions.
  • Blueprint History — all generated designs saved to user account with timestamp, client name, and wreath size.
  • Blueprint Reload — reload any past design, regenerate with new inventory, or fork into a new brief.
  • Subscription Tiers (Bloom / Craft) — Bloom = 5 designs/mo free; Craft = unlimited designs + history. Stripe integration.
  • Client Profiles — save client names, past briefs, and order notes for repeat business.

FUTURE Studio Tier

  • ABC Lab (Adaptive Blueprint Composer) — run evolutionary algorithm on blueprint to score and auto-repair balance, cluster count, and style compliance.
  • Blueprint Scoring Dashboard — EC_WR_V2 validator output with visual scoring on: asymmetry, cluster count, layer balance, inventory compliance, style rules.

Technical Foundation
Stack & Architecture
Phase 1 Stack (Ship Now)
  • Frontend: Next.js 14 App Router + React + Tailwind CSS
  • AI: Claude Sonnet 4 via Anthropic API — server-side only, never NEXT_PUBLIC_
  • Blueprint Canvas: SVG (hand-rolled, no lib required for v1)
  • Output: JSON download + formatted HTML for copy-paste
  • Hosting: Vercel (zero config, free tier for MVP)
  • No auth, no DB, no Stripe — pure pipeline tool in Phase 1
Phase 2+ Stack (Add as Needed)
  • Auth + DB: Supabase (auth, PostgreSQL, RLS, realtime)
  • Payments: Stripe (subscriptions, one-time, marketplace payouts)
  • Marketplace: Stripe Connect for blueprint seller payouts
  • Commerce: Etsy API v3 for direct listing push (Phase 4)
  • Canvas: Canvas API for QACS + Motion Engine (Phase 3–4)
  • Typography: Cormorant Garamond + Dancing Script + Inter + DM Mono
API Design Principles (Non-Negotiable)
  • All Claude API calls are server-side only — API key in ANTHROPIC_API_KEY, never NEXT_PUBLIC_
  • All blueprint outputs validated against EC_WR_V2 schema before returning to client
  • AI system prompts must include "Respond ONLY with valid JSON — no preamble, no markdown, no backticks" for all structured outputs
  • CSS tokens must use --ec-* variables — no raw hex values in component code
  • All tier-gated features use blur overlay with upgrade CTA — never display:none
Risk Register
Known Risks & Mitigations
API Cost at Scale Medium Claude API cost per pipeline run is ~$0.01–0.05 at Sonnet rates. Phase 1 has no auth so abuse is possible. Mitigate: add rate limiting by IP at Vercel edge; require email in Phase 2 to gate usage.
Inventory Hallucination High System prompt must include "NEVER invent inventory" with explicit enforcement. EC_WR_V2 validator cross-checks blueprint elements against structured_inventory before output is returned. Flag mismatches as errors, not warnings.
Etsy SEO Accuracy Medium We generate listing copy, not keyword data. Recommend users cross-reference tags with eRank/Marmalead. Add disclaimer in UI: "Tags are AI-generated — validate with Etsy SEO tools before publishing."
Market Niche Size Low Etsy wreath/faux floral is a multi-hundred-million-dollar vertical. 2M+ active Etsy sellers in home decor. Even 0.1% adoption at $49/mo Studio tier = $1M ARR. Niche is large enough.
Competitor Clone Risk Low The full pipeline (emotion engine + inventory constraints + polar blueprint + faux render rules + genome system) is a multi-layer moat. A competitor could build pieces, but not the integrated system quickly. First mover matters here.
Claude API Breaking Changes Low Always use pinned model string (claude-sonnet-4-20250514 or latest stable). Wrap all API calls in try/catch with graceful degradation. Never expose raw API errors to users.
Definition of Done
Phase 1 Success Criteria
Functional Gates (Must Pass)
  • Pipeline runs end-to-end in under 20 seconds on a cold start
  • Blueprint JSON validates against EC_WR_V2 schema with 0 errors on 10 test inputs
  • Focal cluster count is odd (3, 5, or 7) on 100% of outputs
  • No inventory items appear in blueprint that weren't in the user's input list
  • Render prompt contains at least 3 of the 5 anti-fresh marker phrases
  • Etsy listing outputs exactly 13 tags on every run
  • Blueprint SVG renders correctly on mobile (375px viewport)
Business Gates (30 Days Post-Launch)
  • 25+ unique users generate a design in first 30 days (organic, no paid ads)
  • At least 3 users share output on social media or Etsy community
  • At least 1 user reports using the render prompt in Midjourney and sharing result
  • Zero critical bug reports related to inventory hallucination
  • Net Promoter feedback: "Would you pay $15/mo for unlimited designs?" — 40%+ say Yes
Phase 1 Definition of "Done"
Phase 1 is done when a wreath maker can paste in a client's emotional brief and their inventory list, click one button, and copy a complete design package — blueprint, render prompt, build guide, Etsy listing — in under 30 seconds, with zero hallucinated materials. That's the win. Everything else is Phase 2.
