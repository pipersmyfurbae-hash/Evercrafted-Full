# Floral Emotion Tagger — Developer Spec
**Evercrafted Platform · Craft Tier App**
Version 1.0 · April 2026

---

## 1. What This App Does

The Floral Emotion Tagger takes a raw floral inventory — entered as plain text, a list, or CSV rows — and assigns each item an emotional profile using two inputs: the floral type (what the flower symbolically and psychologically conveys) and its color (how the specific color shifts or amplifies the base emotion). The output is a tagged inventory ready to be consumed by the Blueprint Composition Engine and Emotion-to-Design Translator. A collection summary is produced alongside per-item tags, surfacing the dominant emotional range across the full inventory, identifying emotion gaps, and suggesting design intent.

This app is the entry point for emotion-driven wreath design — every blueprint generated by Evercrafted begins with an emotion-tagged inventory.

---

## 2. User Flow

1. User logs in to Evercrafted (Supabase auth)
2. System reads their tier from `profiles.tier`
3. User navigates to Floral Emotion Tagger (available Craft+)
4. User selects input mode: **Inventory** (multiple items) or **Single Item** (one query)
5. User enters inventory text or selects a quick example
6. User clicks "Tag Emotions"
7. App POSTs to `/api/claude` — server verifies auth + tier before calling Claude
8. Claude returns JSON with per-item tags + collection summary
9. App renders results: item cards with emotion tags + Collection Summary card
10. Session is persisted to `tag_sessions` in Supabase
11. User can export JSON or CSV (CSV export gated to Studio+)
12. User can view session history (Craft+)
13. Studio+ users can upload a bulk CSV file instead of typing
14. Atelier users can access tagged data via API

## 4. AI Integration

### Required Tier
Craft (minimum)

### Model
`claude-sonnet-4-20250514`

### System Prompt
The system prompt is baked into `lib/claude.ts`. It instructs Claude to:
- Parse each inventory item's floral type and color
- Apply the Floral Emotion Reference Table (22 floral types with primary + secondary emotions)
- Apply the Color Emotion Modifier Table (19 color groups)
- Merge type + color into 2–4 emotion tags per item
- Output a collection summary with dominant emotions, emotional range, and gap analysis
- Use gender-neutral language throughout
- Flag low-confidence items
- Exclude cherry blossoms, pussy willow, and twig-based blossoms

### Input Format (sent to `/api/claude`)
```json
{
  "input": "12 stems white peony\n6 dusty rose garden rose\n8 sage eucalyptus",
  "mode": "inventory"
}
```

### Output Format (returned from Claude)
```json
{
  "items": [
    {
      "name": "White Peony",
      "floral_type": "peony",
      "color": "white",
      "category": "focal",
      "quantity": 12,
      "emotion_tags": ["softness", "purity", "grace", "peace"],
      "primary_emotion": "softness",
      "color_modifier": "white amplifies purity and peace, softening all base emotions",
      "confidence": "high"
    }
  ],
  "collection_summary": {
    "dominant_emotions": ["calm", "nostalgia", "softness", "warmth"],
    "emotional_range": "quiet-to-tender",
    "design_intent_suggestion": "Best suited for nostalgic or soft romantic wreath blueprints.",
    "emotion_gaps": ["drama", "boldness"],
    "gap_note": "No high-drama florals. Add burgundy dahlia or deep protea to expand range."
  }
}
```

### Max Tokens
2048 per request

## 5. Data Model

### Table: `profiles`
| Column | Type | Notes |
|---|---|---|
| id | uuid | FK → auth.users.id |
| tier | text | bloom / craft / studio / atelier |
| email | text | From auth on signup |
| created_at | timestamptz | |
| updated_at | timestamptz | |

RLS: Users can only SELECT and UPDATE their own row.

### Table: `tag_sessions`
| Column | Type | Notes |
|---|---|---|
| id | uuid | PK, gen_random_uuid() |
| user_id | uuid | FK → profiles.id |
| input_text | text | Raw user input |
| result | jsonb | Full TaggerResult object |
| label | text | Nullable user label for session |
| created_at | timestamptz | |

RLS: Users can SELECT, INSERT, UPDATE, DELETE their own rows only.

Index: `(user_id, created_at DESC)` for fast history queries.

## 9. Known Limitations / Future Work

**Current Limitations**
- No real-time collaboration (Atelier shared inventory is spec'd but not yet built)
- CSV bulk upload parses client-side before sending as text; proper file streaming is a future enhancement
- No undo/edit for tagged sessions — delete and re-run
- Emotion reference table is static in the system prompt; it will need to update as the Evercrafted floral canon evolves

**Future Work (Roadmap)**
- `/skill/blueprint-feed`: one-click pipeline from tagged inventory → Blueprint Composition Engine
- Inventory library: save and name inventories for reuse across design sessions
- Emotion heatmap: visual grid showing emotion distribution across all tagged sessions
- Supplier SKU enrichment: match tagged items to Evercrafted supplier catalog automatically
- Webhook to Blooms inventory: when a new item is added to Blooms, auto-trigger emotion tagging
- Mobile PWA: offline mode with local-first tagging, sync when connected
